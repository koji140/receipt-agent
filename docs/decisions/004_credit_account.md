# 004 貸方科目の設計

## 決定日
2026-03-20

## 決定内容
貸方科目は「短期借入金」と「普通預金（各口座）」の2択のみとし、
reconcileStatusとは完全に独立して管理する。

## 貸方科目マッピング

| タイミング | 貸方科目 |
|:--|:--|
| 登録時（照合前） | 短期借入金 |
| GMOあおぞら照合後 | 普通預金（GMOあおぞらネット銀行） |
| 東山口信用金庫照合後 | 普通預金（東山口信用金庫） |
| ゆうちょ銀行照合後 | 普通預金（ゆうちょ銀行） |
| 現金払い確定時 | 短期借入金 |
| 個人経費（personal） | 短期借入金 |

## expenseTypeとの関係

| expenseType | 照合前 | 照合後 |
|:--|:--|:--|
| `corporate` | 短期借入金 | 普通預金（照合口座） |
| `personal` | 短期借入金 | 短期借入金（変更なし） |

`expenseType` のデフォルトは `corporate`。`personal` への変更はシート直接編集で行い、
`onEdit` トリガーが変更履歴に記録する。

## 背景・理由
- 領収書だけでは支払元（カード・口座）は基本的に判別できない
- 登録時に毎回支払元を指定する運用は煩雑
- 銀行照合で事後的に確定する方がシンプルで正確

## 重要な注意事項
「仮」「確定」の区別は設けない。貸方科目は単純に「短期借入金」か「普通預金（どの口座か）」の2択。
reconcileStatusを見れば照合済みかどうかは分かるため、貸方科目に状態を持たせる必要はない。

## Script Properties
| キー | 値 |
|:--|:--|
| `CREDIT_CORPORATE` | 短期借入金 |
| `CREDIT_PERSONAL` | 短期借入金 |
| `CREDIT_GMO` | 普通預金（GMOあおぞらネット銀行） |
| `CREDIT_HIGASHI` | 普通預金（東山口信用金庫） |
| `CREDIT_YUCHO` | 普通預金（ゆうちょ銀行） |
