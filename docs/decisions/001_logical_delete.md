# 001 論理削除の採用

## 決定日
2026-03-20

## 決定内容
経費シートのレコード削除は論理削除のみとし、物理削除を禁止する。

## 背景・理由
- 重複判定が誤っていた場合（例：Gensparkのように同月に正当な複数購入があるケース）にデータを復元できる必要がある
- 監査ログとの整合性を維持するために削除後もレコードを追跡できる必要がある
- 誤削除のリバートが容易であること

## 実装方針
経費シートに以下の列を追加する。

| 列名 | 型 | 説明 |
|:--|:--|:--|
| `is_deleted` | boolean | 論理削除フラグ |
| `deleted_at` | datetime | 削除日時 |
| `deleted_by` | string | 削除者 |
| `delete_reason` | string | 削除理由 |

## 運用ルール
- 論理削除された行は集計・照合・グローバルレビューの対象外とする
- 監査ログ・変更履歴には全件記録する
- 誤削除時は `is_deleted=false` に戻すことで復元する
- 論理削除されたDriveファイルは `DRIVE_DELETED_FOLDER_ID` フォルダに移動する

## 月割り行の連動削除
親行（年額サブスク）を論理削除する際は、AIが月割り子行の状況を分析して確認メッセージを提示する。

| 月割り行の状態 | AIの推奨アクション |
|:--|:--|
| 計上済み（payment_date確定済み） | 残すことを推奨 |
| 未来月（未計上） | 削除を推奨 |
| 当月（計上予定） | 個別確認を促す |

## 却下した選択肢
- **物理削除**: 元に戻せないため却下
