# 002 照合ステータス5段階管理

## 決定日
2026-03-20

## 決定内容
銀行照合のステータスを5段階で管理する。

## ステータス定義

| ステータス | 意味 | 貸方科目 |
|:--|:--|:--|
| `unreconciled` | 未照合（登録時デフォルト） | 短期借入金 |
| `reconciled` | 照合確定済み | 普通預金（照合口座）or 短期借入金 |
| `reconciled_modified` | 照合後に手動修正あり | 同上 |
| `cash_payment` | 現金払い・照合不要と確定 | 短期借入金 |
| `reconcile_pending` | 照合待ち（通帳待ち等） | 短期借入金 |

## 背景・理由
当初は `unreconciled` / `reconciled` の2段階を想定していたが、以下のケースへの対応が必要と判断した。

- **永久に照合されないケースがある**: 現金払い・個人立替は銀行データに出てこない
- **通帳待ちのケースがある**: 東山口信用金庫・ゆうちょはオンライン照合不可のため通帳写真待ちになる
- **照合後に修正が発生するケースがある**: 支払日の誤り・口座の間違いなど

## 貸方科目の設計
現金払い・個人カード払いを問わず貸方科目は**短期借入金で統一**する。照合の有無だけで貸方科目を分けず、銀行照合で法人口座が確認された場合のみ普通預金に更新する。

## 自動処理
`unreconciled` が30日以上経過した行は月末レビュー時にAIが「現金払いでしたか？」と確認する。

## 却下した選択肢
- **2段階管理**: 永久未照合ケースに対応できないため却下
- **現金払いで貸方科目を分ける**: シンプルさを優先して却下
