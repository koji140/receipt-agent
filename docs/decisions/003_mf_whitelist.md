# 003 MFホワイトリスト＋照合済みID方式

## 決定日
2026-03-20

## 決定内容
MoneyForwardデータの照合・漏れ検知にホワイトリスト方式と照合済みID管理を組み合わせる。

## 背景・理由
MoneyForwardには経費に関係ある取引と個人的な取引が大量に混在している。
実際のデータを確認したところ、食費・タクシー・Amazon日用品・ATM引き出し・ローン返済など
経費外の取引が多数あり、全件を「登録漏れ候補」として提示するのは現実的でない。

## 実装方針

### ホワイトリストシート（MFホワイトリスト）
経費になりうる取引先・キーワードを登録し、ヒットした取引のみを照合対象とする。

| 列名 | 説明 |
|:--|:--|
| `whitelistId` | ID |
| `keyword` | マッチキーワード |
| `match_target` | 内容 or 大カテゴリ |
| `estimated_account` | 推定勘定科目 |
| `note` | 備考 |

### 照合済みID管理
経費シートに `mf_transaction_id` 列を追加し、照合済みのMF取引IDを記録する。
ホワイトリストにヒットしたが経費シートに対応行がない取引を「漏れ候補」として提示する。

## 照合フロー

    MFデータ取り込み
        ↓
    ホワイトリストでフィルタリング
    → ヒットしない取引は完全無視
        ↓
    経費シートと照合（金額＋取引先＋日付±1日）
        ↓
    照合成功 → mf_transaction_id記録・payment_date確定
    照合失敗 → 全体レビューアラートに記録
        ↓
    月末レビュー時にAIが提示

## ユーザーフィードバック
「不要」と回答した取引パターンは学習ログに記録し、次回以降スキップする。

## 却下した選択肢
- **全件提示**: 個人取引が大量にあるため現実的でない
- **カテゴリフィルターのみ**: キーワードとの組み合わせの方が精度が高い
